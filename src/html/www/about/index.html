About me

Hey! Name's Jake. I'm currently a Computer Science major at Oklahoma State, looking to graduate by May 2017. I've always been the type to take interest in things with flashing lights and bright colors, as long as I get to put it together myself. I began programming when I was 18 — creating games in frameworks such as libGDX or XNA — and gained an interest in web design somewhere along the way. Nowadays, you can find me working on front-end UI, game engine architecture, or playing around with a Raspberry Pi. My language of choice right now is probably TypeScript, although I'm not afraid to take the plunge into a C/C++ codebase.

Other interests of mine include drawing, hiking, watching football, and admiring planes as they fly overhead. I like keeping myself active by participating in mud-runs or playing rugby, but I really just like doing anything where I get an excuse to run. Walking on the beach with my girlfriend is what I dream of doing everyday, and a good bowl of pasta will always put me in a great mood.

About this site

This webpage is all about learning! Sure, using a framework such as Bootstrap or Foundation would have made the development of this site much easier, but where's the fun in that? I've had a much better time trying to figure out things like "rems" or vertical margin collapsing by myself1. Sarcasm aside, this site is my personal tool for learning the nitty-gritty details of web development. Experimental technologies are always welcome here, and will be used with reckless abandon, so don't be too surprised when WebAssembly becomes the backbone of this page! I try not to break the site too much, but it happens every so often.

In the spirit of learning, I've created a few "guidelines" for this site to abide by. These rules don't necessarily facilitate quick development or guarantee a polished final product, but they instead allow me to make mistakes and learn important lessons from them. If any of the following rules are broken in any way, please file an issue here. The rules are as follows:

1. JavaScript is optional

JavaScript is a great2 tool for adding additional features to any webpage. Unfortunately, across the web, JS has also been involved in many, many, many issues relating to user-friendliness. Understandably, concerned users have reacted by disabling JS in their browsers. To avoid serving broken webpages to these users, JS usage is limited towards enhancements only (e.g., AJAX). Anything considered "necessary" for the site to work should not rely on JS.

2. No WCMS

Web Content Management Systems, such as WordPress, are very popular tools for generating and managing content on a webpage. While this type of tool can be extremely useful, and should probably be used for anything serious, I am choosing to do most things manually here. Along with the lessons learned from this development style, hopefully this will give me a better appreciation for WCMS-type tools in the future.
